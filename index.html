Privacy Policy

Effective Date: February 2026

This Privacy Policy describes how StrickerJoy Studios handles information within the CryptFile application. Our core philosophy is simple: your data belongs to you, and we have no interest in seeing it.

1. Comprehensive No Data Collection Policy

CryptFile is built to function entirely without the need for data collection. We do not collect, store, transmit, or share any of the following:

  • Personal Information: We do not ask for names, emails, or phone numbers.
  • Cryptographic Data: We never see your passwords, salt values, or encryption keys.
  • File Content: We do not have access to the files you select to encrypt or decrypt.
  • Usage Data: We do not use trackers, cookies, or telemetry to monitor how you use the app.

2. Localized Data Processing

The security of your data is maintained by keeping it local. All encryption and decryption logic is executed 100% on the user's device hardware. At no point is unencrypted or encrypted data sent to an external server or cloud provider for processing. This ensures that your files remain private even when your device is offline.

3. Third-Party Library Integration

CryptFile utilizes the Zip4j open-source library to handle standard ZIP operations and AES encryption. This library is embedded within the application code and operates locally. It does not communicate with external entities or collect any metadata from your device.

4. Storage Access Framework (SAF)

To provide maximum security on modern operating systems, CryptFile uses the Storage Access Framework (SAF). This ensures the app operates with "Least Privilege" principles. The app cannot see your entire storage; it only gains permission to interact with specific files and folders that you manually select through the system file picker.
